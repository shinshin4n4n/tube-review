# Claude CLI プロンプト管理

このディレクトリは、Claude CLI に渡す指示文（プロンプト）を管理します。

## 目的

- プロンプトの再利用性を高める
- 作業履歴を追跡可能にする
- 成功/失敗したプロンプトから学ぶ

## ファイル命名規則
```
{連番}_{作業内容}.md
```

例:
- `01_setup_docs.md` - ドキュメント初期セットアップ
- `02_database_setup.md` - データベース設計
- `03_auth_implementation.md` - 認証機能実装

## プロンプトファイルの構成

各プロンプトファイルは以下の構成で作成：
```markdown
# プロンプト: {タイトル}

## 目的
{このプロンプトが何を達成するか}

## 実行日
YYYY-MM-DD

## 指示内容
{Claude CLI に渡す具体的な指示}

## 期待される成果物
- ファイル1
- ファイル2

## 備考
{追加情報、注意点など}
```

## 使い方

### 1. プロンプトファイルを読む
```bash
cat .claude/prompts/01_setup_docs.md
```

### 2. Claude CLI で実行

プロンプトの内容をコピーして：
```bash
claude
# プロンプトを貼り付けて実行
```

または、ファイルから直接読み込み（対応している場合）：
```bash
claude < .claude/prompts/01_setup_docs.md
```

### 3. 結果を確認してコミット
```bash
git add .
git commit -m "docs: Add project documentation"
```

## プロンプトの改善

うまくいかなかった場合：
1. プロンプトファイルを修正
2. コミットメッセージに「修正理由」を記載
3. 再実行

Git 履歴で「どう改善したか」が追える。
