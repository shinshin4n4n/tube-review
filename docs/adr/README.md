# Architecture Decision Records (ADR)

## ADRとは

アーキテクチャ上の重要な決定を記録し、その理由を明文化するドキュメント。

## 目的

- 将来の自分や他の開発者が「なぜこうなっているか」を理解できる
- 技術選択の背景を共有できる
- 同じ議論を繰り返さない

## フォーマット

各ADRは以下の構造で記述：

```markdown
# ADR-XXX: タイトル

## ステータス
[提案中 | 承認 | 却下 | 廃止]

## 決定日
YYYY-MM-DD

## コンテキスト
（どんな状況で、なぜこの決定が必要になったか）

## 決定内容
（何を決めたか）

## 理由
（なぜこの選択をしたか）

## 代替案と却下理由
（検討した他の選択肢と、なぜ採用しなかったか）

## 結果・影響
（この決定による影響、制約、今後の方針）

## 参考資料
（関連ドキュメント、記事、議論のリンク）
```
